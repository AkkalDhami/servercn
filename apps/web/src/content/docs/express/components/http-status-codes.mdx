# HTTP Status Codes

The **HTTP Status Codes** component provides a **centralized and type-safe** way to reference HTTP status codes across your backend.

---

## Installation Guide

Install the component using the servercn CLI:

<PackageManagerTabs command="npx servercn-cli add http-status-codes" />

---

## Why This Component Exists

In many backend codebases, HTTP status codes are often scattered and hard-coded, making the application harder to maintain and reason about:

```ts
res.status(200).json(...);
res.status(401).send(...);
res.status(500).json(...);
```

---

## Basic Implementation


```ts title="src/constants/status-codes.ts"
export const STATUS_CODES = {
  // 2xx Success
  OK: 200,
  CREATED: 201,
  ACCEPTED: 202,
  NO_CONTENT: 204,

  // 3xx Redirection
  MOVED_PERMANENTLY: 301,
  FOUND: 302,
  NOT_MODIFIED: 304,

  // 4xx Client Errors
  BAD_REQUEST: 400,
  UNAUTHORIZED: 401,
  FORBIDDEN: 403,
  NOT_FOUND: 404,
  CONFLICT: 409,
  UNPROCESSABLE_ENTITY: 422,
  TOO_MANY_REQUESTS: 429,

  // 5xx Server Errors
  INTERNAL_SERVER_ERROR: 500,
  NOT_IMPLEMENTED: 501,
  BAD_GATEWAY: 502,
  SERVICE_UNAVAILABLE: 503,
  GATEWAY_TIMEOUT: 504
} as const;

export type StatusCode = (typeof STATUS_CODES)[keyof typeof STATUS_CODES];
```

---

## Usage Example

```ts
import { STATUS_CODES } from "../utils/status-codes";

res.status(STATUS_CODES.OK).json(...);
res.status(STATUS_CODES.BAD_REQUEST).send(...);
res.status(STATUS_CODES.INTERNAL_SERVER_ERROR).json(...);
```

The **HTTP Status Codes** component solves this by providing a clear, standardized, and type-safe way to reference status codes everywhere in your application.

## Status Code Reference

### 2xx Success

These codes indicate that the client's request was successfully received, understood, and accepted.

<Table>
<THead>
<TR>
  <TH>Status Code</TH>
  <TH>Constant</TH>
  <TH>Description</TH>
  <TH>Recommended Usage</TH>
</TR>
</THead>
<TBody>
<TR>
  <TD><Endpoint>200</Endpoint></TD>
  <TD><Endpoint>OK</Endpoint></TD>
  <TD>The request was successful and the response contains the requested data.</TD>
  <TD>Use for successful <Endpoint>GET</Endpoint> requests and <Endpoint>PUT</Endpoint>/ <Endpoint>PATCH</Endpoint>/ <Endpoint>DELETE</Endpoint> requests that return a response body.</TD>
</TR>

<TR>
  <TD><Endpoint>201</Endpoint></TD>
  <TD><Endpoint>CREATED</Endpoint></TD>
  <TD>A new resource was successfully created.</TD>
  <TD>Use after a successful <Endpoint>POST</Endpoint> request that creates a resource. Include the new resource or its identifier in the response.</TD>
</TR>

<TR>
  <TD><Endpoint>202</Endpoint></TD>
  <TD><Endpoint>ACCEPTED</Endpoint></TD>
  <TD>The request has been accepted for asynchronous processing.</TD>
  <TD>Use when the request is queued or handled by a background job and processing is not yet complete.</TD>
</TR>

<TR>
  <TD><Endpoint>204</Endpoint></TD>
  <TD><Endpoint>NO_CONTENT</Endpoint></TD>
  <TD>The request was successful but there is no response body.</TD>
  <TD>Use for successful <Endpoint>DELETE</Endpoint> operations or updates where no data needs to be returned.</TD>
</TR>

</TBody>

</Table>

### 3xx Redirection

These codes indicate that the client must take additional action to complete the request.


<Table>
<THead>
<TR>
  <TH>Status Code</TH>
  <TH>Constant</TH>
  <TH>Description</TH>
  <TH>Recommended Usage</TH>
</TR>
</THead>
<TBody>
<TR>
  <TD><Endpoint>301</Endpoint></TD>
  <TD><Endpoint>MOVED_PERMANENTLY</Endpoint></TD>
  <TD>The resource has been permanently moved to a new URL.</TD>
  <TD>Use when an endpoint has been permanently replaced. Clients should update stored URLs.</TD>
</TR>

<TR>
  <TD><Endpoint>302</Endpoint></TD>
  <TD><Endpoint>FOUND</Endpoint></TD>
  <TD>The resource is temporarily available at a different URL.</TD>
  <TD>Use for temporary redirects such as authentication or OAuth flows.</TD>
</TR>

<TR>
  <TD><Endpoint>304</Endpoint></TD>
  <TD><Endpoint>NOT_MODIFIED</Endpoint></TD>
  <TD>The resource has not changed since the last request.</TD>
  <TD>Use for caching mechanisms with <Endpoint>ETag</Endpoint> or <Endpoint>Last-Modified</Endpoint> headers to reduce bandwidth usage.</TD>
</TR>

</TBody>

</Table>

### 4xx Client Errors

These codes indicate that the request contains bad syntax or cannot be fulfilled.

<Table>
<THead>
<TR>
  <TH>Status Code</TH>
  <TH>Constant</TH>
  <TH>Description</TH>
  <TH>Recommended Usage</TH>
</TR>
</THead>

<TBody>
<TR>
  <TD><Endpoint>400</Endpoint></TD>
  <TD><Endpoint>BAD_REQUEST</Endpoint></TD>
  <TD>The request is malformed or contains invalid parameters.</TD>
  <TD>Use for generic validation errors or malformed request payloads.</TD>
</TR>

<TR>
  <TD><Endpoint>401</Endpoint></TD>
  <TD><Endpoint>UNAUTHORIZED</Endpoint></TD>
  <TD>Authentication is required or the provided credentials are invalid.</TD>
  <TD>Use when access tokens are missing, expired, or invalid.</TD>
</TR>

<TR>
  <TD><Endpoint>403</Endpoint></TD>
  <TD><Endpoint>FORBIDDEN</Endpoint></TD>
  <TD>The client is authenticated but lacks sufficient permissions.</TD>
  <TD>Use for authorization failures (e.g., RBAC or ownership checks).</TD>
</TR>

<TR>
  <TD><Endpoint>404</Endpoint></TD>
  <TD><Endpoint>NOT_FOUND</Endpoint></TD>
  <TD>The requested resource does not exist.</TD>
  <TD>Use when a resource ID, endpoint, or entity cannot be found.</TD>
</TR>

<TR>
  <TD><Endpoint>409</Endpoint></TD>
  <TD><Endpoint>CONFLICT</Endpoint></TD>
  <TD>The request conflicts with the current state of the resource.</TD>
  <TD>Use for duplicate resources (e.g., email already exists) or state conflicts.</TD>
</TR>

<TR>
  <TD><Endpoint>422</Endpoint></TD>
  <TD><Endpoint>UNPROCESSABLE_ENTITY</Endpoint></TD>
  <TD>The request is valid but fails domain-specific validation rules.</TD>
  <TD>Use for semantic validation errors (e.g., weak password, invalid state transition).</TD>
</TR>

<TR>
  <TD><Endpoint>429</Endpoint></TD>
  <TD><Endpoint>TOO_MANY_REQUESTS</Endpoint></TD>
  <TD>Too many requests have been sent in a given time window.</TD>
  <TD>Use when rate limits are exceeded. Include retry headers when possible.</TD>
</TR>

</TBody>
</Table>

---

### 5xx Server Errors

These codes indicate the server failed to fulfill an apparently valid request.


<Table>
<THead>
<TR>
  <TH>Status Code</TH>
  <TH>Constant</TH>
  <TH>Description</TH>
  <TH>Recommended Usage</TH>
</TR>
</THead>

<TBody>
<TR>
  <TD><Endpoint>500</Endpoint></TD>
  <TD><Endpoint>INTERNAL_SERVER_ERROR</Endpoint></TD>
  <TD>An unexpected server error occurred.</TD>
  <TD>Use as a fallback for unhandled exceptions or unknown failures.</TD>
</TR>

<TR>
  <TD><Endpoint>501</Endpoint></TD>
  <TD><Endpoint>NOT_IMPLEMENTED</Endpoint></TD>
  <TD>The requested functionality is not supported.</TD>
  <TD>Use for unimplemented endpoints or feature placeholders.</TD>
</TR>

<TR>
  <TD><Endpoint>502</Endpoint></TD>
  <TD><Endpoint>BAD_GATEWAY</Endpoint></TD>
  <TD>An invalid response was received from an upstream service.</TD>
  <TD>Use when a dependent service returns an invalid response.</TD>
</TR>

<TR>
  <TD><Endpoint>503</Endpoint></TD>
  <TD><Endpoint>SERVICE_UNAVAILABLE</Endpoint></TD>
  <TD>The service is temporarily unavailable.</TD>
  <TD>Use during maintenance windows or when the system is overloaded.</TD>
</TR>

<TR>
  <TD><Endpoint>504</Endpoint></TD>
  <TD><Endpoint>GATEWAY_TIMEOUT</Endpoint></TD>
  <TD>An upstream service failed to respond in time.</TD>
  <TD>Use when timeouts occur while waiting for dependent services.</TD>
</TR> 
</TBody>
</Table>
