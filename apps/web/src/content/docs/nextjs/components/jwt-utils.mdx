# JWT Utils Function

## Overview

The <Code>jwt-utils</Code> component is a small wrapper around <Code children="jsonwebtoken" /> that standardizes:

- **Token shapes** (typed access + refresh payloads)
- **Signing** with fixed expirations
- **Verification** with consistent error types
- **Header parsing** for <Code children="Authorization: Bearer <token>" />


## Installation Guide

Install the component using the servercn CLI:

<PackageManagerTabs command="npx servercn-cli add jwt-utils" />

## Environment variables

These helpers read secrets from <Code children="process.env" />:

- <Code children="JWT_ACCESS_SECRET" />: secret for access tokens
- <Code children="JWT_REFRESH_SECRET" />: secret for refresh tokens

## Token types & payloads

The module defines a discriminated union via <Code children="type" />:

- <Code children="JwtType.ACCESS" /> with <Code children="AccessTokenPayload" />
- <Code children="JwtType.REFRESH" /> with <Code children="RefreshTokenPayload" />

Minimal payload fields:

- **Access**: <Code children="sub" />, <Code children="type" />, <Code children="email" />, <Code children="sid" />
- **Refresh**: <Code children="sub" />, <Code children="type" />, <Code children="sid" />

<Code children="iat" />/<Code children="exp" /> are optional fields that come from JWT signing.

## Signing tokens

### <Code children="signAccessToken(payload)" />

Signs an access token with:

- **Secret**: <Code children="process.env.JWT_ACCESS_SECRET" />
- **Expires**: **15m**
- **Payload**: <Code children='{ sub, type: "access", email, sid }' />

### <Code children="signRefreshToken(payload)" />

Signs a refresh token with:

- **Secret**: <Code children="process.env.JWT_REFRESH_SECRET" />
- **Expires**: **7d**
- **Payload**: <Code children='{ sub, type: "refresh", sid }' />

Example:

```ts
import { signAccessToken, signRefreshToken } from "@/lib/jwt";

const access = signAccessToken({ sub: user.id, email: user.email, sid: session.id });
const refresh = signRefreshToken({ sub: user.id, sid: session.id });
```

## Verifying tokens

### 1. verifyAccessToken(token)

Verifies an access token and returns an <Code children="AccessTokenPayload" />.

### 2. verifyRefreshToken(token)

Verifies a refresh token and returns a <Code children="RefreshTokenPayload" />.

### Errors

Verification throws **typed errors** you can catch:

- <Code children="JwtExpiredError" />: token expired (maps from <Code children="jsonwebtoken" />'s <Code children="TokenExpiredError" />)
- <Code children="JwtInvalidError" />: invalid token (signature/format/etc.)

Example (Next.js Route Handler):

```ts
import { NextResponse } from "next/server";
import {
  extractTokenFromHeader,
  verifyAccessToken,
  JwtExpiredError,
  JwtInvalidError
} from "@/lib/jwt";

export async function GET(req: Request) {
  const token = extractTokenFromHeader(req.headers.get("authorization") ?? undefined);
  if (!token) return NextResponse.json({ error: "Missing bearer token" }, { status: 401 });

  try {
    const payload = verifyAccessToken(token);
    return NextResponse.json({ userId: payload.sub, email: payload.email, sid: payload.sid });
  } catch (err) {
    if (err instanceof JwtExpiredError) {
      return NextResponse.json({ error: "Access token expired" }, { status: 401 });
    }
    if (err instanceof JwtInvalidError) {
      return NextResponse.json({ error: "Invalid access token" }, { status: 401 });
    }
    return NextResponse.json({ error: "Auth error" }, { status: 401 });
  }
}
```

import { decodeToken } from "@/lib/jwt";
