# Installation

This guide will help you install and set up ServerCN in your Node.js project.

## Prerequisites

Before installing ServerCN, ensure you have:

- **Node.js** version 18 or higher
- **npm**, **yarn**, **bun** or **pnpm** package manager
- An existing Node.js project (or create a new with servercn foundations)

## Quick Start

The fastest way to get started with ServerCN:

### 1. Initialize ServerCN in your project

<PackageManagerTabs command="npx servercn init" />

### 2. Add your first component

<PackageManagerTabs command="npx servercn add error-handler" />

## Configuration Options

**Project Root Directory**

- Default: <Code children="." /> (current directory)
- The root directory where your project lives

**Source Directory**

- Default: <Code children="src" />
- Where your source code will be located

**Architecture**

- **MVC**: Traditional Model-View-Controller structure
- **Feature-based**: Feature-module architecture

**Language**

- **TypeScript**: for type safety

**Backend Framework**

- **Express**: Currently supported framework

**Database**

- **MongoDB**: With Mongoose ODM
- **MySQL**: With Drizzle ORM
- **PostgreSQL**: With Drizzle ORM
- More databases coming soon

## servercn.config.json

This file stores your project configuration and is used by ServerCN to generate components in the correct structure.

```json
{
  "version": "1.0.0",
  "project": {
    "root": ".",
    "srcDir": "src",
    "type": "backend"
  },
  "stack": {
    "runtime": "node",
    "language": "typescript",
    "framework": "express",
    "architecture": "feature"
  },
  "database": {
    "type": "mongodb",
    "orm": "mongoose"
  },
  "overrides": {},
  "meta": {
    "createdAt": "2026-01-17T12:09:31.721Z",
    "createdBy": "servercn@1.0.0"
  }
}
```

## Databases

ServerCN provides first-class support for popular databases through well-integrated ORMs and ODMs. This ensures your project follows best practices for data modeling and type safety.

### MongoDB with Mongoose

ServerCN supports **MongoDB** using the **Mongoose ODM**. When you choose this option, the CLI will:

- Set up a robust database connection utility.
- Scaffold Mongoose schemas and models following your chosen architecture (MVC or Feature).
- Handle common Mongoose patterns for authentication and role-based access control.

### MySQL with Drizzle

For **MySQL**, ServerCN uses **Drizzle ORM**. This setup provides:

- Fully type-safe schemas using Drizzle's TypeScript-first approach.
- Migration scripts and configuration for `drizzle-kit`.
- Efficient relational data modeling with Drizzle's declarative syntax.

### PostgreSQL with Drizzle

ServerCN also supports **PostgreSQL** via **Drizzle ORM**, offering:

- Advanced PostgreSQL features mapping directly to Drizzle schemas.
- Type-safe queries and relations.
- Seamless integration with PostgreSQL-specific data types.

## Adding Components

Once initialized, you can add components to your project:

```bash
# Add a component
npx servercn add <component-name>

# Examples
npx servercn add error-handler

# Add multiple components
npx servercn add jwt-utils http-status-codes
```

## Project Structure

After initialization, your project structure will look like this:

```
my-backend/
├── src/                    # Source directory
├── servercn.config.json          # ServerCN configuration
├── package.json           # Node.js dependencies
└── tsconfig.json          # TypeScript configuration
```

### MVC Architecture

When using MVC architecture, components are organized as:

```
src/
├── configs/               # Configuration files
├── constants/            # Constants and enums
├── controllers/          # Route controllers
├── middlewares/          # Express middlewares
├── models/               # Database models
├── routes/               # Route definitions
├── services/             # Business logic
├── utils/                # Utility functions
└── validations/          # Validation schemas
```

### Feature-Based Architecture

When using feature-based architecture, components are organized as:

```
src/
├── shared/               # Shared code
│   ├── configs/
│   ├── constants/
│   ├── errors/
│   ├── middlewares/
│   └── utils/
├── modules/              # Feature modules
│   └── user/
│       ├── user.controllers.ts
│       ├── user.routes.ts
│       ├── user.services.ts
│       └── user.validations.ts
└── routes/               # Main route index
```

## Installing Components

When you add a component, ServerCN will:

1. **Copy component files** to your project following your architecture
2. **Install dependencies** required by the component
3. **Update <Code children=".env.example" />** with required environment variables
4. **Create necessary directories** if they don't exist

### Example: Adding Error Handler

<PackageManagerTabs command="npx servercn add error-handler" />

This will:

- Create **src/middlewares/error-handler.ts** (MVC) or **src/shared/middlewares/error-handler.ts** (Feature)
- Create **src/utils/api-error.ts** or **src/shared/errors/api-error.ts**
- Create **src/constants/status-codes.ts** or **src/shared/constants/status-coes.ts**
- Install any required npm packages

## Updating Configuration

You can manually edit <Code children="servercn.config.json" /> to change your project configuration:

```json
{
  "stack": {
    "architecture": "feature" // Change from "mvc" to "feature"
  }
}
```

After updating, new components will use the updated configuration.

## Verifying Installation

To verify ServerCN is properly installed:

1. Check for <Code children="servercn.config.json" /> in your project root
2. Run <Code children="npx servercn list" /> to see available components
3. Try adding a component: <Code children="npx servercn add error-handler" />

## Troubleshooting

### Already Initialized

If you see "ServerCN is already initialized":

- Your project already has a <Code children="servercn.config.json" /> file
- You can start adding components immediately
- To reinitialize, delete <Code children="servercn.config.json" /> first

### Component Not Found

If a component doesn't exist:

```bash
# List all available components
npx servercn list

# Check component name spelling
npx servercn add error-handler  # ✅ Correct
npx servercn add errorhandler   # ❌ Incorrect
```

### Architecture Mismatch

If components aren't appearing in the expected location:

1. Check your <Code children="servercn.config.json" /> architecture setting
2. Verify your <Code children="srcDir" /> configuration
3. Ensure directories exist before adding components

## Uninstalling

To remove ServerCN from your project:

1. Delete <Code children="servercn.config.json" /> configuration file
2. Remove any ServerCN components you've added
3. Remove ServerCN-specific dependencies (if any)
