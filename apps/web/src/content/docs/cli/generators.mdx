# Generators

Generators allow you to quickly scaffold boilerplate code for your application.

Instead of manually creating files, you can use the CLI:

<PackageManagerTabs command="npx servercn-cli g <type> <name> [options]" />

## Options

- <Code>--force</Code>:  Overwrite existing files
- <Code>--flat</Code>:   Generate files in a flat structure
- <Code>-h</Code>, <Code>--help</Code>: display help for command

## Available Generators

### Controller

<PackageManagerTabs command="npx servercn-cli g co auth" />

#### Examples:

##### 1. MVC Architecture
```
> npx servercn-cli g co auth

CREATE: src\controllers\auth.controller.ts

✔ Successfully generated controller: auth
```
```ts title="src/controllers/auth.controller.ts"
import { Request, Response } from "express";

class AuthController {
  async test(req: Request, res: Response) {
    res.status(200).json({
      success: true,
      message: "Controller added successfully"
    });
  }
}

export const authController = new AuthController();
```

---

##### 2. Feature Architecture
```
> npx servercn-cli g co auth

CREATE: src\modules\auth\auth.controller.ts

✔ Successfully generated controller: auth
```
```ts title="src/modules/auth/auth.controller.ts"
import { Request, Response } from "express";

class AuthController {
  async test(req: Request, res: Response) {
    res.status(200).json({
      success: true,
      message: "Controller added successfully"
    });
  }
}

export const authController = new AuthController();
```

### Model

<PackageManagerTabs command="npx servercn-cli g mo user" />

#### Examples:

##### 1. MVC Architecture

```ts title="src/models/user.model.ts"
import mongoose, { Document, Model, Schema } from "mongoose";

export interface IUser extends Document {
  _id: mongoose.Types.ObjectId;

  name: string;

  createdAt: Date;
  updatedAt: Date;
}

const userSchema = new Schema<IUser>(
  {
    name: {
      type: String,
      required: true,
      trim: true
    }
  },
  {
    timestamps: true
  }
);

const User: Model<IUser> =
  mongoose.models.User || mongoose.model<IUser>("User", userSchema);

export default User;
```

---

##### 2. Feature Architecture


### DTO

Generate **Data Transfer Objects (DTOs)** using <Code>zod</Code> for validation and type inference.

DTOs are used to validate incoming data (e.g. request body, params, query).

<PackageManagerTabs command="npx servercn-cli g dto <name> [fields...] [options]" />

**fields:** 
Define fields using: <Code>key:type</Code>

<PackageManagerTabs command="npx servercn-cli g dto auth" />

<PackageManagerTabs command="npx servercn-cli g dt user --flat" />

<PackageManagerTabs command="npx servercn-cli g dt user name:string email:email age:number" />

##### Creates:

- MVC: <Code>src/dtos/auth.dto.ts</Code>
- Feature: <Code>src/modules/auth/auth.dto.ts</Code>

## Help

<PackageManagerTabs command="npx servercn-cli g --help" />

```
Usage: servercn-cli generate|g [options] <type> <name> [fields...]

Generate resources (controller, service, resource, model,
route, dto, validator, etc.)

Options:
  --force     Overwrite existing files
  --flat      Generate files in a flat structure
  -h, --help  display help for command

Aliases:
  co → controller
  se → service
  mo → model
  ro → route
  re → resource
  dt → dto
  va → validator

Examples:
  $ npx servercn-cli g co user
  $ npx servercn-cli g re auth 
  $ npx servercn-cli g mo user
  $ npx servercn-cli g ro user
  $ npx servercn-cli g se user
  $ npx servercn-cli g dt user
  $ npx servercn-cli g va user
```
